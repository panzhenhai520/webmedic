# 阶段 5：音频上传与 ASR 转写链路

## 5.1 目标

完成“医生说话 / 患者说话”按钮录音上传，并返回转写结果显示到页面。

## 5.2 本阶段必须完成的内容

### 前端

* 使用 MediaRecorder 录音
* 支持“医生说话”开始/结束录音
* 支持“患者说话”开始/结束录音
* 上传音频片段到后端
* 显示返回的转写结果

### 后端

* 接收音频文件上传
* 保存音频文件到 uploads
* 建立 transcript_segments 记录
* 先做 ASR 适配层 `asr_service.py`
* 第一版可以先 mock 转写结果
* 保留未来切换 Dolphin 的封装口

## 5.3 本阶段不要做的事

* 不要做自动说话人分离
* 不要做实时 token 级流式
* 不要复杂化音频切片逻辑

## 5.4 Claude 本阶段输出文件建议

* `backend/app/api/asr.py`
* `backend/app/services/asr_service.py`
* `backend/app/services/transcript_service.py`
* `frontend/src/composables/useRecorder.js`
* `frontend/src/api/asr.js`
* 更新工作站组件

## 5.5 完成标准

* 点击“医生说话”可录音并上传
* 点击“患者说话”可录音并上传
* 页面显示转写文本
* 数据库存入 transcript_segments 记录
