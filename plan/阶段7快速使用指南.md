# 阶段7快速使用指南

## 使用步骤

### 1. 准备病历文件

在 `D:\webmedic\medical_records\` 目录下放置一些 PDF 文件（用于测试）。

如果没有真实病历，可以创建几个简单的 PDF 文件：
- 可以用 Word 创建文档，然后另存为 PDF
- 或者下载任意 PDF 文件重命名为 case_001.pdf, case_002.pdf 等

### 2. 启动服务

**后端：**
```bash
cd D:\webmedic\backend
venv\Scripts\activate
python run.py
```

**前端：**
```bash
cd D:\webmedic\frontend
npm run dev
```

### 3. 操作流程

访问 http://localhost:5173，按以下顺序操作：

#### 第一步：扫描病历文件
1. 在左侧控制面板找到"病历管理"部分
2. 点击"扫描本地病历"按钮
3. 等待扫描完成，会显示发现的文件数量

#### 第二步：重建索引
1. 点击"重建索引"按钮
2. 等待索引完成

#### 第三步：开始问诊
1. 点击"开始问诊"
2. 点击"医生说话"和"患者说话"录入对话
   - 建议多录几次，覆盖主诉、现病史、体格检查、诊断等信息
3. 点击"重新抽取"执行结构化抽取
4. 切换到右侧"结构化病历"标签页查看抽取结果

#### 第四步：检索相似病历
1. 点击"检索相似病历"按钮
2. 切换到右侧"相似病历"标签页查看结果
3. 会显示 3 条相似病历，包含：
   - 文件名
   - 相似度分数
   - 匹配原因

## 常见问题

### Q: 点击"检索相似病历"提示"没有已索引的病历文档"
**A:** 需要先完成以下步骤：
1. 点击"扫描本地病历"
2. 点击"重建索引"

### Q: 点击"检索相似病历"提示"没有结构化记录"
**A:** 需要先：
1. 开始问诊
2. 录入对话
3. 点击"重新抽取"

### Q: 扫描病历失败
**A:** 检查：
1. `D:\webmedic\medical_records\` 目录是否存在
2. 目录中是否有 PDF 文件
3. 后端服务是否正常运行

### Q: 相似病历结果是随机的？
**A:** 是的，当前版本使用 Mock 实现，返回随机结果。这是为了演示功能流程。未来接入 PageIndex 后会进行真实的语义检索。
