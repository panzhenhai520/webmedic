# WebMedic 阶段8完成说明

## 本阶段实现内容

### 后端实现

1. **Schema 定义**
   - `app/schemas/draft_schema.py` - 病历草稿 Schema
   - `app/schemas/clinical_hint_schema.py` - 临床提示 Schema

2. **Prompt 模板**
   - `app/prompts/generate_emr_draft.txt` - 病历草稿生成 Prompt
   - `app/prompts/generate_clinical_hints.txt` - 临床提示生成 Prompt

3. **服务层**
   - `app/services/draft_service.py` - 病历草稿生成服务
     - Mock 模式：基于结构化记录生成草稿
     - LLM 模式：调用 DeepSeek 生成草稿
     - 应用相似病历方案功能
   - `app/services/clinical_hint_service.py` - 临床提示生成服务
     - Mock 模式：基于规则生成提示
     - LLM 模式：调用 DeepSeek 生成提示
     - 生成风险提示、追问建议、建议检查

4. **API 接口**
   - `app/api/endpoints/draft.py`
     - `POST /api/v1/draft/generate/{session_id}` - 生成病历草稿
     - `POST /api/v1/draft/apply-similar-plan/{session_id}` - 应用相似方案
   - `app/api/endpoints/clinical_hints.py`
     - `POST /api/v1/clinical-hints/generate/{session_id}` - 生成临床提示

5. **数据库落库**
   - `emr_drafts` 表 - 保存病历草稿
   - `clinical_hints` 表 - 保存临床提示

### 前端实现

1. **API 封装**
   - `src/api/draft.js` - 病历草稿 API
   - `src/api/clinicalHints.js` - 临床提示 API

2. **组件更新**
   - `src/components/workbench/DraftView.vue` - 病历草稿展示组件（新增）
   - `src/components/workbench/ControlPanel.vue` - 添加生成草稿和提示功能
   - `src/components/workbench/AssistantTabs.vue` - 添加病历草稿标签页

3. **状态管理**
   - `src/stores/workbench.js` - 添加草稿状态管理

## 测试步骤

### 完整流程测试

1. **准备工作**
   - 访问 http://localhost:5173
   - 确保已上传或扫描了病历文件

2. **开始问诊**
   - 点击"开始问诊"
   - 点击"医生说话"和"患者说话"录入对话
   - 建议多录几次，覆盖主诉、现病史、体格检查、诊断等信息

3. **结构化抽取**
   - 点击"重新抽取"
   - 切换到"结构化病历"标签页查看结果

4. **检索相似病历**
   - 点击"检索相似病历"
   - 切换到"相似病历"标签页查看结果

5. **生成病历草稿**
   - 点击"生成病历草稿"
   - 切换到"病历草稿"标签页查看结果
   - 应该看到完整的病历草稿，包含：
     - 主诉、现病史、既往史、过敏史
     - 体格检查、初步诊断
     - 建议检查、治疗方案

6. **生成临床提示**
   - 点击"生成提示"
   - 切换到"风险提示"标签页查看风险提示
   - 切换到"追问建议"标签页查看追问问题
   - 应该看到：
     - 风险提示（如药物过敏提示、跌倒风险等）
     - 追问建议（如询问既往病史、血压等）
     - 建议检查（如颈椎X光、血常规等）

### 预期结果

**病历草稿：**
- 显示"AI生成草稿，仅供医生参考"警告
- 包含完整的病历字段
- 治疗方案使用建议性语言
- 数据保存到 `emr_drafts` 表

**临床提示：**

## 注意事项

1. **医疗安全**
   - 所有AI生成内容标注"仅供参考"
   - 使用建议性语言，避免确定性表述
   - 不输出"必须用某药"等强制性内容

2. **Mock 数据**
   - 当前版本使用 Mock 生成，结果为规则生成
   - 真实场景需接入 DeepSeek API
   - Prompt 模板可根据实际需求调整

3. **性能考虑**
   - LLM 调用可能需要几秒钟
   - 建议添加加载状态提示
   - 考虑实现缓存机制

4. **数据隐私**
   - 病历数据包含敏感信息
   - 生产环境需要加密存储
   - API 调用需要脱敏处理

## 故障排查

### 生成草稿失败提示"没有结构化记录"

- 先执行"重新抽取"操作
- 确保有对话记录

### 生成提示失败

- 检查是否已执行结构化抽取
- 查看后端日志确认错误原因

### 草稿内容显示不完整

- 检查 Mock 数据生成逻辑
- 查看数据库 `emr_drafts` 表的 `content_json` 字段

### LLM 模式调用失败

- 检查 `.env` 中的 `DEEPSEEK_API_KEY` 是否正确
- 检查网络连接
- 查看后端日志确认错误信息
