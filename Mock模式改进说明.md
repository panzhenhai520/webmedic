# Mock 模式改进说明

## 改进内容

将 Mock 模式从"返回固定数据"改为"基于对话内容提取"，确保逻辑一致性。

## 改进前 vs 改进后

### 改进前（固定数据）

```python
# 完全忽略对话内容，返回硬编码数据
return {
    "chief_complaint": "颈部疼痛伴头晕3天",
    "present_illness": "患者3天前无明显诱因出现颈部疼痛...",
    ...
}
```

**问题**：
- 与实际对话内容无关
- 无法用于真实功能测试
- 逻辑不一致

### 改进后（基于对话内容）

```python
# 从对话中提取关键信息
- 提取症状：痛、晕、恶心等
- 提取部位：颈部、头部、腰部等
- 提取时间：3天、5天、一周等
- 提取过敏史：青霉素、头孢等
- 提取既往史：高血压、糖尿病等
```

**优点**：
- 基于实际对话内容
- 逻辑一致
- 可用于功能测试
- 不需要调用真实 API

## 提取逻辑

### 1. 主诉提取

**提取内容**：
- 症状：痛、晕、恶心、咳嗽、发烧等
- 部位：头、颈、腰、腹、胸等
- 时间：X天、X日

**示例**：
```
对话：患者：我颈椎痛，已经六天了
提取：主诉 = "颈椎疼痛6天"
```

### 2. 现病史提取

**构建规则**：
```
患者 + [时间] + 出现 + [部位] + [症状] + [其他描述]
```

**示例**：
```
对话：患者：颈椎痛六天了，越来越严重
提取：现病史 = "患者6天前出现颈椎疼痛，症状逐渐加重。"
```

### 3. 时间提取

**支持格式**：
- 阿拉伯数字：3天、5天、10天
- 中文数字：三天、五天、十天
- 取最后一次提到的时间

**示例**：
```
对话：
  患者：三天了
  患者：不对，是五天
  患者：应该是六天吧

提取：时间 = 6天（取最后一次）
```

### 4. 过敏史提取

**关键词**：过敏、不能吃、不能用

**常见过敏原**：青霉素、头孢、磺胺、海鲜、花生等

**示例**：
```
对话：患者：我对青霉素过敏
提取：过敏史 = "对青霉素过敏"
```

### 5. 既往史提取

**常见疾病**：高血压、糖尿病、心脏病、肝炎、肾病

**示例**：
```
对话：患者：我有高血压
提取：既往史 = "既往有高血压"
```

### 6. 初步诊断

**基于症状和部位推断**：
- 颈部 + 疼痛 → 颈椎病
- 头部 + 疼痛 → 头痛待查
- 腹部 + 疼痛 → 腹痛待查

### 7. 建议检查

**基于初步诊断**：
- 颈椎病 → 颈椎X光片或CT、血常规
- 头痛 → 头颅CT、血常规
- 其他 → 相关影像学检查、血常规

### 8. 风险标记

**提取规则**：
- 有过敏史 → 注意XX过敏
- 有头晕症状 → 注意防跌倒

## 优势与限制

### 优势

1. **逻辑一致**：提取结果基于实际对话内容
2. **可测试性**：可用于功能测试，验证提取逻辑
3. **零成本**：不需要调用真实 API
4. **快速响应**：提取速度快（<0.1秒）
5. **可预测**：基于规则，结果可预测

### 限制

1. **简单规则**：只能进行关键词匹配，无法理解复杂语义
2. **有限覆盖**：只支持常见症状和部位
3. **无推理能力**：无法像真实 LLM 那样进行推理
4. **中文依赖**：主要针对中文对话

### Mock vs 真实 LLM

| 特性 | Mock 模式 | 真实 LLM |
|------|----------|----------|
| 数据来源 | 对话内容（关键词） | 对话内容（语义理解） |
| 提取准确性 | 中等（70-80%） | 高（90-95%） |
| 语义理解 | 无 | 强 |
| 推理能力 | 无 | 强 |
| 响应速度 | 极快（<0.1秒） | 较慢（2-5秒） |
| 成本 | 免费 | 消耗 API 额度 |
| 适用场景 | 功能测试、演示 | 生产使用 |
