# 智能录音模式说明

## 工作原理

系统现在使用**智能 fallback 机制**，自动选择最优录音方式：

```
开始录音
    ↓
检查浏览器是否支持 WebM
    ↓
    ├─ 支持 → 使用 WebM 模式（优先）
    │         - MediaRecorder API
    │         - 文件小（压缩格式）
    │         - 前端处理简单
    │         - 后端 ffmpeg 转换
    │
    └─ 不支持 → 使用 WAV 模式（Fallback）
              - Web Audio API
              - 文件大（未压缩）
              - 前端编码处理
              - 后端直接识别
```

## 模式对比

### WebM 模式（优先）✅

**优点**：
- ✅ 文件小（约 50KB/秒，压缩格式）
- ✅ 上传快（网络传输时间短）
- ✅ 前端简单（浏览器原生支持）
- ✅ CPU 占用低（硬件编码）

**缺点**：
- ⚠️ 需要后端 ffmpeg 转换（增加 1-2 秒）
- ⚠️ 依赖 ffmpeg 安装

**适用场景**：
- 生产环境（ffmpeg 已安装）
- 网络较慢的环境
- 长时间录音

### WAV 模式（Fallback）

**优点**：
- ✅ 无需后端转换（直接识别）
- ✅ 无需 ffmpeg（降低依赖）
- ✅ 格式标准（兼容性好）

**缺点**：
- ⚠️ 文件大（约 250KB/秒，未压缩）
- ⚠️ 上传慢（网络传输时间长）
- ⚠️ 前端处理复杂（需要编码）
- ⚠️ CPU 占用高（软件编码）

**适用场景**：
- ffmpeg 不可用
- 老旧浏览器
- 短时间录音

## 性能对比

### 10 秒录音示例

| 指标 | WebM 模式 | WAV 模式 |
|------|-----------|----------|
| 文件大小 | ~50KB | ~250KB |
| 上传时间（1Mbps） | ~0.4秒 | ~2秒 |
| 前端处理 | 0秒 | ~0.5秒 |
| 后端转换 | ~1秒 | 0秒 |
| **总耗时** | **~1.4秒** | **~2.5秒** |

### 30 秒录音示例

| 指标 | WebM 模式 | WAV 模式 |
|------|-----------|----------|
| 文件大小 | ~150KB | ~750KB |
| 上传时间（1Mbps） | ~1.2秒 | ~6秒 |
| 前端处理 | 0秒 | ~1.5秒 |
| 后端转换 | ~2秒 | 0秒 |
| **总耗时** | **~3.2秒** | **~7.5秒** |

**结论**：WebM 模式在大多数情况下更快！

## 自动切换逻辑

系统会自动检测并选择最优模式：

1. **检查浏览器支持**：
   ```javascript
   if (MediaRecorder.isTypeSupported('audio/webm')) {
     // 使用 WebM 模式
   } else {
     // 使用 WAV 模式
   }
   ```

2. **WebM 失败自动切换**：
   - 如果 WebM 录音失败
   - 自动切换到 WAV 模式
   - 用户无感知

3. **后端智能处理**：
   - 收到 `.webm` → 使用 ffmpeg 转换
   - 收到 `.wav` → 直接识别
   - ffmpeg 不可用 → 返回明确错误

## 浏览器兼容性

### 支持 WebM（使用 WebM 模式）
- ✅ Chrome 49+
- ✅ Edge 79+
- ✅ Firefox 25+
- ✅ Opera 36+

### 不支持 WebM（使用 WAV 模式）
- ⚠️ Safari（部分版本）
- ⚠️ IE（所有版本）
- ⚠️ 老旧浏览器

## 配置要求

### WebM 模式要求
- ✅ 现代浏览器（Chrome/Firefox/Edge）
- ✅ 后端安装 ffmpeg
- ✅ ffmpeg 在系统 PATH 中

### WAV 模式要求
- ✅ 支持 Web Audio API 的浏览器
- ✅ 无需 ffmpeg

## 故障排查

### WebM 模式失败
**症状**：自动切换到 WAV 模式

**原因**：
- 浏览器不支持 WebM
- MediaRecorder API 不可用

**解决**：
- 使用现代浏览器
- 或接受 WAV 模式（仍可正常工作）

### 后端转换失败
**症状**：错误提示 "ffmpeg 未安装"

**原因**：
- ffmpeg 未安装
- ffmpeg 未在 PATH 中

**解决**：
1. 运行 `check_ffmpeg.bat` 检查状态
2. 运行 `setup_ffmpeg.bat` 安装 ffmpeg
3. 重启 Dolphin ASR 服务

### 两种模式都失败
**症状**：无法录音

**原因**：
- 麦克风权限未授予
- 浏览器版本过旧

**解决**：
- 检查浏览器麦克风权限
- 更新浏览器到最新版本
