# 医疗词库维护指南

## 概述

`medical_vocabulary.py` 是 Mock 模式下使用的医疗词汇库。当系统配置为 `LLM_USE_MOCK=true` 时，Instructor 和 LangExtract 两个抽取器都会使用这个词库进行关键词匹配。

## 词库结构

### 1. BODY_PARTS（身体部位）
存储身体部位的标准名称和各种说法。

**格式：**
```python
"标准名称": ["关键词1", "关键词2", "关键词3"]
```

**示例：**
```python
"肱骨": ["肱骨"],
"膝关节": ["膝关节", "膝盖"],
```

### 2. DIRECTIONS（方位词）
存储方位描述词。

**示例：**
```python
"左": ["左", "左侧", "左边"],
"右": ["右", "右侧", "右边"],
```

### 3. SYMPTOMS（症状）
存储症状的标准名称和各种说法。

**示例：**
```python
"疼痛": ["痛", "疼", "疼痛", "酸痛", "刺痛"],
```

### 4. DISEASES（疾病/诊断）
存储常见疾病和诊断。

**示例：**
```python
"胆囊息肉": ["胆囊息肉"],
"颈椎病": ["颈椎病"],
```

## 如何添加新词汇

### 场景1：添加骨科专业词汇

**需求：** 患者说"我的肱骨骨折了"，希望能识别"肱骨"。

**操作：**
1. 打开 `medical_vocabulary.py`
2. 找到 `BODY_PARTS` 字典
3. 在骨科专业部分添加：
```python
# 骨科专业 - 骨骼
"肱骨": ["肱骨"],
"桡骨": ["桡骨"],
"尺骨": ["尺骨"],
```

### 场景2：添加内科疾病

**需求：** 患者说"我有胆囊息肉"，希望能识别这个诊断。

**操作：**
1. 打开 `medical_vocabulary.py`
2. 找到 `DISEASES` 字典
3. 在消化系统部分添加：
```python
# 消化系统
"胆囊息肉": ["胆囊息肉"],
"肝囊肿": ["肝囊肿"],
```

### 场景3：添加症状的多种说法

**需求：** 患者可能说"睡不好"、"失眠"、"睡眠不好"，都应该识别为失眠。

**操作：**
```python
"失眠": ["失眠", "睡不着", "睡眠不好", "睡不好", "入睡困难"],
```

## 维护原则

### 1. 标准化命名
- Key 使用医学标准术语
- 例如：用"肱骨"而不是"大臂骨"

### 2. 包含口语化表达
- Value 列表中包含患者可能使用的口语说法
- 例如："膝关节" → ["膝关节", "膝盖"]

### 3. 避免重复
- 检查是否已存在相同或相似的词条
- 例如："颈部"和"颈椎"应该分开

### 4. 按科室分类
- 使用注释标记不同科室的词汇
- 便于后期维护和扩展

### 5. 考虑方位组合
- 系统会自动组合方位词和部位词
- 例如："左" + "肱骨" → "左肱骨"

## 词库的局限性

即使维护了完整的词库，Mock 模式仍有局限：

1. **无法理解语义**
   - 只能做字符串匹配
   - 无法理解复杂的医学描述

2. **无法推理**
   - 无法根据症状推断可能的诊断
   - 无法理解症状之间的关联

3. **维护成本高**
   - 需要持续添加新词汇
   - 医学术语数量庞大

## 建议

对于生产环境，强烈建议：
1. 关闭 Mock 模式（`LLM_USE_MOCK=false`）
2. 使用真实的 LLM API（DeepSeek）
3. LLM 具备完整的医学知识，无需维护词库

Mock 模式适用于：
- 开发测试环境
- 完全离线环境
- 零成本要求的场景
