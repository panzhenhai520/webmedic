## 对话记录
{dialogue_text}

## 患者基本信息
- 姓名：{patient_name}
- 性别：{gender}
- 年龄：{age}岁

## 任务要求
请仔细分析对话内容，**严格基于对话中明确提到的信息**进行提取。如果某项信息在对话中未提及，请填写"未提及"或留空。

**重要原则：**
1. **不要推测或编造信息** - 只提取对话中明确提到的内容
2. **不要添加医学常识** - 即使某些症状通常伴随某些体征，如果对话中没提到，就不要写
3. **不要做诊断推断** - 除非医生在对话中明确给出诊断，否则填写"待查"
4. **体格检查** - 只记录对话中医生明确提到的检查结果，没有就填"未提及"
5. **治疗方案** - 只记录对话中医生明确建议的检查或治疗，没有就填"未提及"

## 输出格式
你必须输出一个有效的 JSON 对象，包含以下字段：

```json
{{
  "chief_complaint": "主诉内容（患者主要不适症状的简要描述，包括部位、症状、时间）",
  "present_illness": "现病史（详细描述患者当前疾病的发生、发展过程，仅基于对话中提到的内容）",
  "past_history": "既往史（仅记录对话中提到的过去疾病史、手术史等，没提到填'无特殊'或'未提及'）",
  "allergy_history": "过敏史（仅记录对话中明确提到的过敏史，没提到填'无'）",
  "physical_exam": "体格检查（仅记录对话中医生明确提到的体征和检查结果，没提到填'未提及'）",
  "preliminary_diagnosis": "初步诊断（仅记录对话中医生明确给出的诊断，没有填'待查'）",
  "suggested_exams": "建议检查（仅记录对话中医生明确建议的检查项目，没有填'未提及'）",
  "warning_flags": "风险标记（仅记录对话中提到的风险因素、禁忌症等，没有填'无'）"
}}
```

## 注意事项
1. 必须输出有效的 JSON 格式
2. 所有字段值必须是字符串类型
3. **严格遵守"只提取、不推测"原则**
4. 主诉应包含：部位 + 症状 + 时间（如"颈部疼痛、头晕3天"）
5. 现病史应完整记录患者描述的症状发展过程
6. 如果对话中没有提到体格检查结果，physical_exam 必须填"未提及"
7. 如果对话中没有给出明确诊断，preliminary_diagnosis 必须填"待查"
8. 如果对话中没有建议检查，suggested_exams 必须填"未提及"

请直接输出 JSON 对象，不要包含任何其他文字说明。
