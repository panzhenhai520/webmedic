# 风险提示显示问题分析与修复

## 问题描述

点击"生成临床提示"按钮后，右侧风险提示标签页显示两行空行，有感叹号图标，但没有任何文字内容。

## 问题原因

**数据结构不匹配**：后端返回的数据字段名与前端期望的字段名不一致。

### 后端返回的数据结构

```javascript
{
  warnings: [
    {
      title: "药物过敏提示",           // ← 字段名是 title
      content: "患者对青霉素过敏...",  // ← 字段名是 content
      severity: "warn"
    }
  ]
}
```

### 前端期望的数据结构

```javascript
{
  warnings: [
    {
      hint_title: "药物过敏提示",      // ← 期望字段名是 hint_title
      hint_content: "患者对青霉素过敏...", // ← 期望字段名是 hint_content
      severity: "warn"
    }
  ]
}
```

### 前端显示代码

在 `AssistantTabs.vue` 第 118-120 行：

```vue
<el-alert
  :title="hint.hint_title"      <!-- 使用 hint_title -->
  :description="hint.hint_content"  <!-- 使用 hint_content -->
  :type="getHintType(hint.severity)"
/>
```

因为字段名不匹配，`hint.hint_title` 和 `hint.hint_content` 都是 `undefined`，所以只显示图标，没有文字。

## 风险提示的内容和依据

### 风险提示包含什么内容？

风险提示（warnings）主要包括：

1. **药物过敏提示**
   - 依据：结构化病历中的"过敏史"字段
   - 示例：如果患者对青霉素过敏，会提示"用药时需避免使用青霉素类抗生素"

2. **跌倒风险提示**
   - 依据：结构化病历中的"主诉"字段
   - 示例：如果患者有头晕、眩晕症状，会提示"注意防跌倒，避免高空作业和驾驶"

3. **一般提示**
   - 依据：如果没有特殊风险，会显示一般性建议
   - 示例："建议根据检查结果制定治疗方案，注意观察病情变化"

### 风险提示的生成逻辑

**位置**：`backend/app/services/clinical_hint_service.py`

**Mock 模式生成逻辑**（第 118-167 行）：

```python
# 1. 检查过敏史
if "青霉素" in structured_record.allergy_history:
    warnings.append({
        "title": "药物过敏提示",
        "content": "患者对青霉素过敏，用药时需避免使用青霉素类抗生素",
        "severity": "warn"
    })

# 2. 检查症状
if "头晕" in structured_record.chief_complaint or "眩晕" in structured_record.chief_complaint:
    warnings.append({
        "title": "需注意跌倒风险",
        "content": "患者有头晕症状，建议注意防跌倒，避免高空作业和驾驶",
        "severity": "warn"
    })

# 3. 默认提示（如果没有特殊风险）
if not warnings:
    warnings.append({
        "title": "一般提示",
        "content": "建议根据检查结果制定治疗方案，注意观察病情变化",
        "severity": "info"
    })
```

**LLM 模式生成逻辑**（第 182-230 行）：

使用 DeepSeek API 根据结构化病历和相似病历生成智能提示。

## 修复方案

### 方案1：修改后端返回数据（推荐）

修改 `backend/app/services/clinical_hint_service.py`，将字段名改为前端期望的格式。

**修改位置**：第 120-124 行、第 128-132 行、第 164-167 行

```python
# 修改前
warnings.append({
    "title": "药物过敏提示",
    "content": "患者对青霉素过敏...",
    "severity": "warn"
})

# 修改后
warnings.append({
    "hint_title": "药物过敏提示",
    "hint_content": "患者对青霉素过敏...",
    "severity": "warn"
})
```

### 方案2：修改前端显示代码

修改 `frontend/src/components/workbench/AssistantTabs.vue`，使用后端返回的字段名。

**修改位置**：第 118-120 行

```vue
<!-- 修改前 -->
<el-alert
  :title="hint.hint_title"
  :description="hint.hint_content"
  :type="getHintType(hint.severity)"
/>

<!-- 修改后 -->
<el-alert
  :title="hint.title"
  :description="hint.content"
  :type="getHintType(hint.severity)"
/>
```

---

## 推荐修复方案

**推荐使用方案1（修改后端）**，原因：

1. **数据库字段名一致性**：数据库模型使用 `hint_title` 和 `hint_content`
2. **API 一致性**：其他地方可能也期望这个字段名
3. **减少前端改动**：只需修改后端一处

---

## 其他临床提示功能

除了风险提示，系统还生成：

1. **追问建议**（followup_questions）
   - 依据：结构化病历中缺失的信息
   - 示例："是否有高血压、糖尿病等慢性病史？"

2. **建议检查**（suggested_exams）
   - 依据：初步诊断
   - 示例："颈椎X光片或CT"、"血常规"

这些数据也需要确保字段名一致。
