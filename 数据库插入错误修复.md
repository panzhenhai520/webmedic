# 数据库插入错误修复

## 错误信息

```
Column 'hint_content' cannot be null
```

## 问题原因

在修改字段名后，保存到数据库的代码还在使用旧的字段名（`"title"`, `"content"`）来获取值，导致获取到 `None`，而数据库要求 `hint_content` 不能为 null。

## 修复内容

修改 `backend/app/services/clinical_hint_service.py` 第 67-68 行：

```python
# 修改前
hint_title=warning.get("title"),      # 获取不到值，返回 None
hint_content=warning.get("content"),  # 获取不到值，返回 None

# 修改后
hint_title=warning.get("hint_title"),    # 正确获取值
hint_content=warning.get("hint_content"), # 正确获取值
```
