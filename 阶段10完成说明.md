## 使用说明

### 1. 环境准备

```bash
# 安装依赖
cd D:\webmedic\backend
pip install -r requirements.txt

# 配置环境变量
# 编辑 .env 文件，添加以下配置：
VECTOR_DB_TYPE=qdrant
QDRANT_MODE=embedded
QDRANT_PATH=./qdrant_storage
QDRANT_COLLECTION=medical_cases
QDRANT_VECTOR_SIZE=1024

EMBEDDING_MODEL=BAAI/bge-m3
EMBEDDING_DEVICE=cpu
EMBEDDING_BATCH_SIZE=32

EXTRACTOR_TYPE=instructor

# 执行数据库迁移
python scripts/add_extractor_type_field.py
```

### 2. 启动服务

```bash
# 启动后端（首次启动会下载BGE-M3模型，约2GB）
cd D:\webmedic\backend
python run.py

# 启动前端
cd D:\webmedic\frontend
npm run dev
```

## 性能指标

### 向量检索性能
- **索引速度**：约1-2秒/文档（包含向量生成）
- **检索速度**：<100ms（10个文档集合）
- **相似度阈值**：0.7（推荐值）
- **返回结果数**：5个（默认）

### 嵌入模型性能
- **模型大小**：约2GB
- **向量维度**：1024
- **编码速度**：约50-100文本/秒（CPU）
- **内存占用**：约2GB（模型加载后）

## 注意事项

### 1. 首次启动
- BGE-M3模型首次加载需要下载约2GB数据
- 下载时间取决于网络速度（5-15分钟）
- 模型缓存在 `~/.cache/huggingface/`

### 2. 内存要求
- 最低：4GB RAM
- 推荐：8GB RAM
- BGE-M3模型占用约2GB内存

### 3. 存储要求
- BGE-M3模型：约2GB
- Qdrant存储：取决于文档数量
- 每个文档向量：约4KB（1024维float32）

### 4. LangExtract
- 当前为可选依赖（注释状态）
- 如需使用，取消注释并安装：`pip install langextract==0.1.5`
- 需要DeepSeek API密钥

## 已知限制

1. **LangExtract未完全测试**：由于langextract库可能不稳定，当前仅实现接口，未深度测试
2. **GPU支持**：需要CUDA环境，当前默认使用CPU
3. **批量索引**：大量文档索引时建议分批处理
4. **向量维度固定**：BGE-M3固定1024维，不可修改
