## 2. FFmpeg 安装脚本说明

### 文件用途

**setup_ffmpeg.bat** 和 **setup_ffmpeg.ps1** 是**必要的工具脚本**，不是调试产生的临时文件。

### 为什么需要这些脚本？

1. **Dolphin ASR 依赖 FFmpeg**
   - Dolphin 需要 FFmpeg 来处理浏览器录音生成的 WebM 格式音频
   - 没有 FFmpeg，Dolphin 无法识别 WebM 格式

2. **简化安装流程**
   - 自动下载 FFmpeg（约 80MB）
   - 自动解压到 C:\ffmpeg
   - 自动添加到系统 PATH
   - 自动验证安装

3. **用户友好**
   - ��用户不需要手动下载和配置
   - 避免常见的安装错误
   - 提供清晰的安装进度提示

### 使用方法

**方法1：使用批处理脚本（推荐）**
```bash
# 右键点击 setup_ffmpeg.bat，选择"以管理员身份运行"
```

**方法2：使用 PowerShell 脚本**
```powershell
# 以管理员身份打开 PowerShell
cd D:\webmedic\backend
.\setup_ffmpeg.ps1
```

### 何时需要运行这些脚本？

1. **首次部署项目**：如果系统中没有安装 FFmpeg
2. **Dolphin ASR 报错**：提示找不到 ffmpeg 或无法处理音频格式
3. **WebM 格式识别失败**：浏览器录音后 Dolphin 无法识别

### 检查 FFmpeg 是否已安装

```bash
# 打开命令行，运行：
ffmpeg -version

# 如果显示版本信息，说明已安装
# 如果提示"不是内部或外部命令"，说明未安装或未添加到 PATH
```

## 建议

### 保留这些脚本

**建议保留** setup_ffmpeg.bat 和 setup_ffmpeg.ps1，原因：

1. **文档化**：这些脚本记录了 FFmpeg 的安装步骤
2. **可重复性**：其他开发者或部署环境可以快速安装 FFmpeg
3. **故障排除**：如果 FFmpeg 出问题，可以重新运行脚本
4. **项目完整性**：这是项目依赖管理的一部分

### 可选：添加到 README

可以在 README.md 中添加说明：

```markdown
## FFmpeg 安装

Dolphin ASR 需要 FFmpeg 来处理音频格式。

### 自动安装（推荐）
右键点击 `backend/setup_ffmpeg.bat`，选择"以管理员身份运行"。

2. 下载：ffmpeg-release-essentials.zip
3. 解压到：C:\ffmpeg
4. 添加到 PATH：C:\ffmpeg\bin
```
