# WebMedic 功能增强完成说明

## 已完成的功能

### 1. 服务端重启问题解决

**问题**：按 Ctrl+C 后无法返回命令行

**解决方案**：
- 创建了 `backend/start.bat` 启动脚本，提供更好的启动和停止体验
- 使用方法：双击 `start.bat` 启动服务，按 Ctrl+C 一次即可停止

**临时解决方法**：
- 按 Ctrl+C 后直接按回车键即可返回命令行
- 或者直接关闭终端窗口重新打开

### 2. 会话历史显示医生和患者信息

**已实现功能**：
- ✅ 会话列表显示医生姓名和职称
- ✅ 会话列表显示患者姓名、性别和年龄
- ✅ 优化表格布局，信息更清晰

**修改的文件**：
- `backend/app/services/session_service.py` - 添加医生和患者信息的关联查询
- `backend/app/schemas/session_history.py` - 更新数据模型
- `backend/app/api/endpoints/sessions.py` - 调整API返回格式
- `frontend/src/views/SessionHistory.vue` - 显示医生和患者信息

**未实现功能**（需要登录系统支持）：
- ⏳ 根据当前登录医生过滤会话历史
- ⏳ 动态医生登录功能

**说明**：
当前系统使用固定的医生（Doctor Panython）和患者（张三），如果需要实现多医生登录和会话过滤功能，需要：
1. 添加用户认证系统（JWT token 或 session）
2. 添加登录/登出接口
3. 在 API 中根据当前登录用户过滤数据
4. 添加权限控制

这是一个较大的功能模块，建议作为独立的阶段（如"阶段10：用户认证与权限管理"）来实现。

## 其他修复

### Pydantic 版本兼容性问题

**问题**：结构化抽取时出现 `argument 'by_alias': 'NoneType' object cannot be converted to 'PyBool'` 错误

**解决方案**：
- 升级 Pydantic 从 2.5.3 到 2.12.5
- 修复了与 OpenAI SDK 2.26.0 的兼容性问题
- 配置 `.env` 文件，设置 `LLM_USE_MOCK=true` 使用模拟模式

## 配置说明

### .env 文件配置

```env
# LLM 服务配置
LLM_USE_MOCK=true  # 设置为 true 使用模拟模式，false 使用真实 DeepSeek API
```

- `LLM_USE_MOCK=true`：使用模拟模式，返回预设的测试数据，不调用真实 API
- `LLM_USE_MOCK=false`：使用真实的 DeepSeek API（需要有效的 API Key）

## 常见问题

### Q1: 服务端按 Ctrl+C 后卡住怎么办？
**A**: 直接按回车键即可返回命令行，或使用新的 `start.bat` 脚本启动服务。

### Q2: PDF无法显示怎么办？
**A**:
1. 确认后端服务正常运行
2. 检查浏览器是否支持PDF查看（Chrome、Edge、Firefox都支持）
3. 检查文件路径是否正确（后端日志会显示）

### Q3: 结构化抽取还是报错怎么办？
**A**:
1. 确认已重启后端服务
2. 检查 `.env` 文件中 `LLM_USE_MOCK=true`
3. 查看后端日志 `backend/logs/error.log`

### Q4: 会话历史为什么显示所有医生的会话？
**A**: 当前系统未实现登录功能，所以显示所有会话。如需按医生过滤，需要实现用户认证系统（建议作为阶段10）。
